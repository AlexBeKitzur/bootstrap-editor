Bootstrap-Editor

A collection of easy-to-setup content editors for Bootstrap.

Editors for every reason.

Many of the projects we work on at Fat Panda involve enabling users to edit and publish content. We love Bootstrap for rapid prototyping, and felt that this ecosystem could benefit from the addition of drop-in content editors, as easy to get running as adding a single <textarea> to your markup.

Currently the goal of this project is to provide editors for three kinds of content: WYSIWYG, Markdown, and source code.

Download

You should download the latest from GitHub. In the archive you will find the following:

./css Stylesheets for all editors, as well as for Font Awesome
./dist All the files you will need to use bootstrap-editor in production
./docs A copy of these docs
./font The font files for Font Awesome
./js The JavaScript libraries
./build-docs.sh A bash script for building and publishing this documentation
./build.sh A bash script for assembling the contents of the ./dist folder

Alternatively, you could add bootstrap-editor to your git project with something like this:

> git submodule add git@github.com:collegeman/bootstrap-editor.git ./htdocs/lib/bootstrap-editor

Setup

In your project create a folder called bootstrap-editor, then copy the folder ./dist from the source archive into your project.

Inside the <head> tag and after the bootstrap stylesheets, add the following:

<link rel="stylesheet" href="/bootstrap-editor/dist/css/font-awesome.min.css">
<link rel="stylesheet" href="/bootstrap-editor/dist/css/bootstrap-editor.min.css">

And before the closing </body> tag and after the bootstrap JavaScripts, add the following:

<!-- if using tinymce: -->
<script src="/bootstrap-editor/dist/js/tinymce/jscripts/tiny_mce/tiny_mce_dev.js"></script>
<!-- if using upload features: -->
<script src="/bootstrap-editor/dist/js/plupload/plupload.js"></script>

<script src="/bootstrap-editor/dist/js/bootstrap-editor.min.js"></script>

WYSIWYG Editor

TinyMCE

Transform a <textarea> input into a TinyMCE editor with formatting tools, drag-and-drop uploading, and distraction-free mode.

<textarea date-edit-as="wysiwyg" placeholder="This is where you write and style!"><textarea>

Usage

Just select an <textarea> using jQuery and call

$('textarea.ed').editor('wysiwyg');

Markup

You will use data attributes to configure your TinyMCE editor. When the DOM is ready, any <textarea> having date-edit-as="wysiwyg" will automatically become a TinyMCE editor.

Methods

$().val()

Use the familiar val() function to read and write from the editor.

<textarea date-edit-as="wysiwyg">foo<textarea>
<script>
  $('textarea.ed').val(); // == <p>foo<p>
  $('textarea.ed').val('<b>bar!</b>'); // HTML editor updated!
</script>

$().editor('exec', cmd)

This will invoke TinyMCE.execCommand(cmd) and so provides you with direct access to the TinyMCE API.

$('textarea.ed').editor('exec', 'bold'); // current selection will toggled bold

Our proxy method accepts all of the parameters accepted by TinyMCE.execCommand():

$('textarea.ed').editor('exec', cmd, ui, val);

PageDown

Transform a <textarea> input into a Markdown editor with preview, drag-and-drop uploading, and distraction-free mode.

<textarea date-edit-as="markdown" placeholder="This is where you write; forget style!"><textarea>
